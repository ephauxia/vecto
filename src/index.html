Playing videos from private Telegram channels

VECTO can stream videos directly from private Telegram channels — including ones that restrict forwarding — without downloading them first. Here's how to set it up.

Step 1 — Get your API credentials

  1. Go to my.telegram.org and log in with your phone number.
  2. Click API Development Tools.
  3. Fill in the app form (name and short name can be anything). Submit.
  4. Note your App api_id and App api_hash. Keep these private — do not share them.

Step 2 — Enter credentials in VECTO

  1. Click the Telegram button in the top bar.
  2. Enter your api_id, api_hash, and the phone number linked to your Telegram account (with country code, e.g. +91xxxxxxxxxx).
  3. Click Save and restart the app when prompted.

Step 3 — Authenticate on first use

  1. After restart, click the Telegram button again.
  2. VECTO will send a login code to your Telegram account (via the official Telegram app or SMS).
  3. Enter the code when prompted. If your account has two-factor authentication enabled, you will also be asked for your password.

Step 4 — Stream a video

  1. In Telegram, right-click (or long-press) a video and copy its link.
  2. Paste the link into the VECTO URL bar and press ADD or Enter.
  3. Playback begins immediately — no download required.

Your credentials are stored securely in your OS keyring and are never sent to any server other than Telegram's own infrastructure.

What is CORS Bypass?

The short version — when would I need this?

If you paste a stream URL into VECTO and it refuses to load with an error like "blocked by CORS policy" or simply fails silently, CORS Bypass may fix it.

This happens most often with streams from servers that haven't been configured to allow playback from outside their own website — for example, streams embedded in a third-party site. Enabling CORS Bypass routes the request through a local proxy running on your machine, which strips the restriction.

The technical version

Browsers enforce the Cross-Origin Resource Sharing (CORS) policy: a page on one origin (e.g. tauri://localhost) cannot fetch resources from a different origin unless the remote server explicitly allows it via the Access-Control-Allow-Origin header.

When CORS Bypass is enabled, VECTO starts a lightweight local HTTP proxy (implemented in Rust, running on 127.0.0.1 on a random port). Stream requests are rewritten to pass through this proxy, which forwards them to the real server and injects the required CORS headers on the response before handing it back to the player. The proxy runs entirely on your machine — no data is routed through any external service.

Should I leave it on all the time?

No. Enable it only when a specific stream fails to load. Leaving it enabled adds a small latency overhead to every request and bypasses a browser security boundary that exists for good reason. For streams that load without it, leave it off.
